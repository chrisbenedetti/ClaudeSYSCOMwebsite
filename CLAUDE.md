## Mission

Build a polished, production-ready SYSCOM corporate website that replaces the current outdated syscom.com. This will be presented to company ownership (Ted Bayer) and EVP (Mark Anzmann) tomorrow morning. The site needs to look like it was designed by a top-tier agency, not generated by AI.

## Context

SYSCOM, Inc. is a 40+ year old Baltimore-based Enterprise Content Management and Business Process Automation firm serving government and financial sector clients. The current website (syscom.com) is outdated, hosted on a legacy provider that the owner describes as "negative value," and does not reflect the company's capabilities or strategic direction.

Three prototype variants were previously built (React/Vite/GitHub Pages):
- **Tech-Forward** — Modern, AI-forward positioning

## What to Build

Build **all three variants** as separate deployable sites in this monorepo, each in its own directory. They should share a common component library where possible but have distinct visual identities. The owner wants to see options side-by-side at the meeting.

### Monorepo Structure
```
/packages/shared/          — Shared components, data, utilities
/packages/techforward/     — Tech-Forward variant
/packages/govfriendly/     — Government-Friendly variant
/packages/premium/         — Premium Dark variant
```

Each variant should be independently buildable and deployable to GitHub Pages.

## Hosting & Deployment

This repo deploys to **two hosts in parallel**:

| Host | What it serves | Why |
|------|----------------|-----|
| **GitHub Pages** | Combined showroom of all 3 variants + samples at `chrisbenedetti.github.io/ClaudeSYSCOMwebsite/` | Internal review tool — lets stakeholders compare variants side-by-side. Stays live even after a variant is chosen. |
| **Cloudflare Pages** | Production deploy of the **Government-Friendly** variant only, at `<project>.pages.dev` (custom domain `syscom.com` deferred) | Production host. Faster CDN, real serverless functions for forms/APIs, PR previews, instant rollback. |

**Do not disable GitHub Pages.** Both deploys trigger from `main`. Full setup is in [DEPLOYMENT.md](./DEPLOYMENT.md).

### `VITE_BASE_PATH` — the conditional base path

`packages/govfriendly/vite.config.ts` reads `process.env.VITE_BASE_PATH` and falls back to `/ClaudeSYSCOMwebsite/govfriendly/` when unset. This lets the same code build correctly for both hosts:

- **GitHub Pages** (no env var set) → asset URLs prefixed with the subpath. Site lives under a project-page subdirectory.
- **Cloudflare Pages** (`VITE_BASE_PATH=/`) → asset URLs at root. Site lives at the root of `*.pages.dev`.

If you change the GH Pages subpath or add a new host, update the fallback or the env var — never hardcode a new base in the config.

### Cloudflare Pages Functions

Server-side endpoints (contact form, future webhooks, etc.) live at `/functions/` at the **repo root** — NOT inside `packages/`. Cloudflare Pages discovers them there regardless of the build output directory.

File-based routing:
- `functions/api/contact.js` → `POST /api/contact`
- `functions/api/[id].js` → dynamic `/api/:id`
- Export `onRequestGet`, `onRequestPost`, etc., or generic `onRequest`.

**Runtime is Cloudflare Workers (Web Fetch API), NOT Node.** No `fs`, no `process`, no `require`. Use `context.env.SECRET_NAME` for env vars and secrets. Secrets are added via `wrangler pages secret put` or the dashboard.

## Design Direction per Variant

### 1. Tech-Forward (`/packages/techforward/`)
**Aesthetic**: Modern, confident, AI-forward. Think Vercel or Stripe's design language — clean but with energy. Subtle animations, gradient accents, technical credibility.
- **Color palette**: Deep navy/slate base, electric blue or teal accents, white text. Subtle gradient meshes.
- **Typography**: Use a distinctive geometric sans-serif for headings (e.g., Outfit, Sora, or Clash Display from Google Fonts). Clean sans for body.
- **Hero**: Bold statement about transforming enterprise content with AI. Animated particle or node network background (subtle, not cheesy). 
- **Tone**: "We've been solving this problem for 40 years. Now we have AI."
- **Key visual motif**: Connected nodes, data flow, transformation

### 2. Government-Friendly (`/packages/govfriendly/`)
**Aesthetic**: Clean, accessible, trustworthy, WCAG AA compliant. Think USDS/18F design system energy — professional government contractor that doesn't look like a government contractor.
- **Color palette**: Navy blue, white, subtle warm gray. Red/burgundy accent sparingly. 
- **Typography**: Readable, professional. Source Sans Pro or similar highly legible font. Generous line height.
- **Hero**: Clear value proposition. No jargon. Emphasize "trusted partner" and track record.
- **Tone**: Reliable, proven, no-nonsense. Emphasize compliance, security, and track record.
- **Key visual motif**: Shield/trust iconography, clean data visualization, structured grids
- **Must have**: High contrast ratios, focus indicators, screen reader friendly, skip navigation

### 3. Premium Dark (`/packages/premium/`)
**Aesthetic**: High-end, sophisticated, exclusive. Think Apple or a luxury brand. Dark theme, dramatic typography, lots of whitespace (darkspace?), cinematic feel.
- **Color palette**: Near-black background (#0a0a0a), off-white text, gold or copper accent
- **Typography**: Elegant serif for headings (e.g., Playfair Display, Cormorant). Refined sans for body.
- **Hero**: Dramatic, minimal. Large typography. "Enterprise Content. Perfected." or similar.
- **Tone**: Quietly confident. Let the work speak. Premium positioning.
- **Key visual motif**: Subtle light effects, refined borders, editorial layout
